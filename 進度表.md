## 📊 專案總覽

把「Gemini 畫繪本 → 取圖片網址 → 產 Apps Script → Google 表單測驗」這條為國小資訊課設計的工作流，包成**一個老師看得懂、學生用得到、AI 全自動的單檔網頁工具**。

| 維度 | 內容 |
|---|---|
| 🎯 受眾 | 國小老師（資訊 / 國語 / 故事教學）|
| 🌐 部署 | GitHub Pages + PWA（離線可用、可裝桌面） |
| 💎 核心價值 | 從**主題打字 → 課堂派發 QR**，全程不離開 App |
| 🆓 成本 | 完全免費（BYOK Gemini Free Tier 1500 RPD） |
| 🛠 技術 | 單檔 HTML + vanilla JS + 3 個 CDN（QRCode / SortableJS / Noto 字型） |

## 🛠 技術架構與設計決策

> 這些是「為什麼選這條路」的紀錄，後人接手 / 自己回頭看時很重要。

### A. 為什麼是單檔 HTML？

- ✅ 老師可 email / USB / Drive 散播，**零安裝**
- ✅ 中小學機房常無管理員權限，**不能裝**任何套件
- ✅ 沒有 build step，老師自己看 view-source 也能修
- ❌ 缺點：檔案大（130 KB），但 PWA 快取後等於**離線也能跑**

### B. 為什麼 BYOK（每人一支自己的 Gemini Key）？

- ✅ 開發者不用負擔 API 費用（每老師有 1500 RPD 免費額度）
- ✅ Key 在使用者瀏覽器內，**永不上傳開發者伺服器**
- ✅ 配額分散，不會「全校用一支 key 互相搶」
- ❌ 缺點：每老師要自己拿 key（30 秒，但是有門檻）
- 用 `x-goog-api-key` header 傳（不放 URL，DevTools Network 看不到）

### C. 為什麼網址用 `/pub` 不用 `/embed`？

- 2024 年前 picconvert 用 `/embed`，**Google 在某時點下架了**
- v0.6.3 我用 curl 實測：`/embed → 404`、`/pub → 200 含 163KB HTML`
- 用 `/pub?embedded=true` 拿到不含 Google 工具列的乾淨版

### D. 為什麼從 `<img alt>` 抓圖片 URL？

- 同一張 Doc 圖片在 HTML 裡有兩個 URL：
  - `alt="https://lh3.googleusercontent.com/..."` ← **可跨域 ✓**
  - `src="https://docs.google.com/docs-images-rt/..."` ← **CORP 擋 ✗**
- 之前 picconvert 兩個都抓 → 我們也跟著抓 → 12 張變 24 張一半破
- v0.6.5 root cause fix：DOM 解析優先讀 alt → 拿到 12 張全可用

### E. AI 圖片如何安全地嵌入 Apps Script？

兩條路徑共存於同一個 Apps Script 生成器：

```
data:image/jpeg;base64,...  →  Utilities.newBlob(Utilities.base64Decode(b64))
                               ↑ AI 生成的圖，不需網路、不會破圖
                               
https://...googleusercontent.com/gg/...  →  UrlFetchApp.fetch(url)
                                            ↑ Doc 抓回來的圖，server-to-server 沒 CORP 限制
```

老師可以**混用兩種來源**（一些 AI 生 + 一些自己畫），生成器自動偵測格式。

### F. PWA 為什麼用 network-first 不用 cache-first？

- HTML / 主 App 用 `network-first`：**有網路一定吃最新版**
- 字型 / CDN / 圖示用 `cache-first`：離線可用
- `version.json` 永遠 `cache: 'no-store'`：自動更新檢查不會被快取
- 配 `skipWaiting + clients.claim` + `controllerchange → reload`：新版部署 30 分內推給老師

### G. 為什麼 0 inline onclick？

- CSP 嚴格設定（`script-src 'self' https://cdn.jsdelivr.net 'unsafe-inline'`）
- 全部用 `addEventListener` + `data-*` 屬性
- 集中在 `wireUI()` 函式管理，找 button 處理函式很快

### 🥇 P1 ─ 高 ROI 候選（任選）

#### P1-1：學生答題分析儀表板　🌟 我最推

**為什麼**：老師最關心「哪題大家答錯」「哪個學生卡關」。目前老師拿到 Google 表單回應只能看原始試算表，沒有統計。

**做法**：
1. Apps Script 加 `onFormSubmit` trigger → 寫進老師指定的 Sheet
2. Sheet → 公開 CSV 連結
3. App 加新分頁「📊 答題分析」→ 拉 CSV → 視覺化
   - 每題答對率 bar chart
   - 學生個別分數
   - 共同錯誤選項（哪題大家最容易選錯）
   - 答題時間分布
4. 用 [Chart.js](https://cdn.jsdelivr.net/npm/chart.js) (CDN, 60KB) 畫圖

**搭配 skill**：`data:build-dashboard`、`data:create-viz`

**工夫**：L（1-2 天）。完成後**閉環就完整了**。

---

#### P1-2：Apps Script API 一鍵部署

**為什麼**：v0.6 的「複製 → 貼到 script.google.com → 執行」對非工程師老師仍是門檻。

**做法**：
1. 用 Google Identity Services (GIS) JS SDK 做 OAuth
2. 拿 token → 呼叫 [Apps Script API](https://developers.google.com/apps-script/api) 建專案 + 跑 buildForm
3. 直接拿到 published form URL → 自動填到步驟 6
4. 老師完全不用碰 script.google.com

**坑**：
- OAuth scope 要 `https://www.googleapis.com/auth/script.projects` + `script.deployments`
- 第一次需要老師授權 N 個權限（教學現場可能猶豫）

**工夫**：XL（2-3 天）。

#### P1-3：批次 AI 出題

**為什麼**：老師可能已經有手寫好的繪本（步驟 3 抓圖），想 AI 一次幫所有章節出題。

**做法**：
- 編輯器加按鈕「✨ 一次幫所有章節出題」

**工夫**：S（1 小時）。**這個價值很高，工夫又少**。

#### P1-5：Cloudflare Workers 代理 Gemini API

**為什麼**：老師擔心 BYOK 暴露 key（雖然 client-side key 沒實際風險）。

**做法**：
- 升級 worker.js 加 Gemini 路由：`?gemini=models/.../generateContent`
- Worker 端用環境變數存 master key（你自己付費）或讓老師存自己的 key 在 Worker
- App 偵測有 `meta.workerUrl` 就走 Worker，沒有走直連

**坑**：
- 配額管理（你的 master key 會被全校老師打）
- 如果改成「老師存自己的 key 在自己的 Worker」就沒這問題

**工夫**：M（半天）。

## 📚 教訓 / Lessons Learned

### 🎯 工程

**1. 第三方 CDN 別 pin 到 patch version**
- `qrcode@1.5.3/build/qrcode.min.js` 在 jsdelivr 神秘 404
- 改 `qrcode@1` 拿最新 1.x 就活了
- patch 版本可能在 CDN 損毀但無人發現

**2. 第三方平台行為會變**
- picconvert 用的 `/embed` Google 在某時點下架
- 寫得時候對的，幾年後可能不對
- 沒看 raw response 就改 code = 浪費時間

**3. 治本 vs 治標的快速判斷**
- v0.6.4 治標（CORS proxy 包裝壞 URL）→ corsproxy 也被擋 → 失敗
- v0.6.5 治本（換 URL 來源）→ 一行 if 解決
- **遇到 bug：先 curl 看 raw → 再寫 fix**

**4. 多版本資料結構升級要寫 migration**
- v0.6.4 → v0.6.5 換抓 URL 邏輯，舊 URL 還躺在 localStorage
- 沒 migration 的話 console 一直噴錯誤
- v0.6.6 補一個 5 行 filter 解決

**5. CSP 嚴格從 v1 就做**
- v0.3 才回頭做 CSP audit，要改 25+ 個 inline onclick
- 一開始就用 `data-action` + `addEventListener`，省 90% 工夫

### 🎓 教學現場洞察

**1. BYOK 比共用 key 健康**
- 老師心理：「我用自己的 key 比較安心」
- 工程上：分散配額、零維運成本

**2. PWA 比想像的重要**
- 學校 WiFi 出名地不穩
- iPad 加入主畫面有「正式 App」感
- 自動更新機制讓老師永遠用新版

**3. 視覺化 > 程式碼**
- 「貼程式碼到 script.google.com」對 95% 老師太抽象
- 視覺化編輯器一出，**整個工具的接受門檻直接降一個檔次**

**4. 第一次造訪的歡迎卡值得做**
- 5 分鐘上手指南放在哪裡都沒人看
- 直接在頁面跳出歡迎卡 → 點擊率 10x

### 🤖 AI 整合

**1. JSON 輸出用 `responseMimeType`**
- `responseMimeType: 'application/json'` 比「請回傳 JSON」可靠 100x
- 還是要寫 `try { JSON.parse(text) }` 防壞掉

**2. 圖像生成有 SAFETY filter**
- 寫實人臉、暴力、特定動作會被擋
- 友善訊息很重要：「試試更柔軟的描述」

**3. 圖直接 base64 內嵌進 Apps Script**
- 比上傳 Drive 拿連結省一個 API 呼叫
- Apps Script `Utilities.newBlob` 吃 base64 沒問題
- 8 張壓縮圖約 1MB，遠低於 Apps Script 10MB 限制
