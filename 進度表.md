## 📊 專案總覽

把「Gemini 畫繪本 → 取圖片網址 → Gemini Canvas 產 Apps Script → Google 表單測驗」這條為國小資訊課設計的工作流，包成一個老師看得懂、學生用得到的單檔網頁工具。

| 維度 | 內容 |
|---|---|
| 受眾 | 國小老師（資訊課 / 國語 / 故事教學）|
| 部署形式 | 單檔 HTML（離線可用）+ 之後可選 GitHub Pages |
| 核心價值 | 把六個分散在不同 Google 服務的步驟串成一條線 |
| 不需要 | 後端、資料庫、付費服務 |

### P0 ─ 馬上做（v0.2，本週）

這幾個做完，工具的「實用性」會跳一個檔次，讓老師真的願意用。

#### 🥇 P0-1：視覺化表單編輯器（解決 B4）

**問題**：現在老師要把 Gemini 給的 code 貼進 script.google.com，或是手改範本的 `pages` 陣列。對非工程師老師來說太抽象。

**解法**：在步驟 5 上方加一個**表格式編輯器**：

```
┌──────────────────────────────────────────────────────┐
│ 章節標題     │ 圖片網址(下拉)│ 故事敘述 │ 題目 │ 選項 │ 正解 │
├──────────────┼─────────────┼──────────┼──────┼──────┼─────┤
│ 第一章 森林   │ [pic 1 ▼]   │ 走進...  │ 往哪 │ A/B/C│ B   │
│ 第二章 河邊   │ [pic 2 ▼]   │ 看見...  │ 是啥 │ A/B/C│ A   │
│ [+ 新增一頁]                                          │
└──────────────────────────────────────────────────────┘
                  ↓ 一鍵產出
        [📋 複製完整 Apps Script]  [💾 存成 .gs]
```

**圖片網址**從步驟 3 抓出來的清單做成下拉選單，免去手貼錯誤。

**工夫**：M（4-6 小時）。完工後**老師可以完全跳過 Gemini Canvas**，直接拿到能跑的 Apps Script。

#### 🥈 P0-2：自架 CORS 代理（解決 B1）

**問題**：corsproxy.io 等第三方一掛，整個步驟 3 廢掉。

**解法**：寫一支 **Cloudflare Workers** 代理（免費 100k 請求/天，對小學一個老師綽綽有餘）：

```js
// worker.js
export default {
  async fetch(request) {
    const url = new URL(request.url).searchParams.get('url');
    if (!url || !url.startsWith('https://docs.google.com/')) {
      return new Response('Bad URL', { status: 400 });
    }
    const res = await fetch(url);
    return new Response(await res.text(), {
      headers: { 'Access-Control-Allow-Origin': '*', 'Content-Type': 'text/html' }
    });
  }
}
```

部署到 `storytell-proxy.<你的帳號>.workers.dev`，把它放在三個第三方代理的最前面當主力，另外三個變備援。

**工夫**：S（1 小時）。**回報無敵高**：穩定、免費、永遠存在。

⚠️ 對應我手上的 skill：`gcp-api-key-secure-create`（如果改用 Google Cloud Functions 也可以；但 Cloudflare Workers 更簡單）。

#### 🥉 P0-3：內建 QR Code 產生器

**問題**：步驟 6 老師拿到表單網址後，要去其他網站產 QR Code 才能在課堂發下去。

**解法**：在步驟 6 加一個輸入框 + 即時 QR：

```html
<input id="formUrl" placeholder="貼上 Google 表單網址">
<canvas id="qrcode"></canvas>
<button>📱 下載 QR (PNG)</button>
<button>🖨 列印派發單 A4</button>
```

用 `qrcode.js` (CDN, 12KB) 直接畫 canvas。**派發單**可以用 `window.print()` + `@media print` CSS 排成漂亮 A4（搭配我手上的 skill `pdf-export-print-best-practice`）。

**工夫**：S（1-2 小時）。

#### P1-4：Web Speech API TTS 試聽

Chrome 內建 `speechSynthesis` 不用任何 API：


#### P1-5：CSP / 安全性 Audit（解決 B2, B6）

- 把所有 inline `onclick=` `onerror=` 改成 `addEventListener`
- `target="_blank"` 全加 `rel="noopener noreferrer"`

- 加 CSP meta tag：`default-src 'self'; img-src https:; connect-src https:`

## 💡 還能延伸的開發方向

### 跨領域整合

- **🧠 NotebookLM 反向接**：把學生答題的回應彙整 → 餵回 NotebookLM → 產出個別化補救建議
- **🎬 動畫繪本**：用 Gemini Veo 生成繪本短片，嵌進表單頭頁當情境引入
- **🎵 配樂引擎**：每章節用 Suno AI 配背景音樂，學生答題時自動播放
- **✏️ 學生創作模式**：反過來給學生這個工具，讓他們自己做繪本表單給同學玩
- **📊 教師社群分享**：做一個「繪本市集」，老師之間分享自己的範本（用 Supabase 存）

### 教學法整合

- **🎯 鷹架式提示**：答錯第一次 hint 簡單；第二次 hint 給線索；第三次直接給答案
- **🎲 隨機題庫**：每個學生看到的題目順序不同，避免抄答案
- **🤝 分組合作**：兩人一組協作答題，加 Google Sheet 同步
- **📝 開放題自動評分**：用 Gemini API 評估開放題答案，給回饋

### 進階技術

- **🔐 短碼分享**：產 6 碼短代碼（例：`STORY-AB12C`），學生輸入就能找到表單
- **📱 LINE Bot 推題**：學生綁定 LINE → 每天推一題故事問題（接 `line-messaging-firebase`）
- **🔍 OCR 入手**：老師拍實體繪本 → 自動 OCR 文字 + 截取插圖 → 跳過 Gemini 直接做表單
- **🎮 闖關地圖**：把線性章節變成地圖式探險，每答對解鎖下一個地點
